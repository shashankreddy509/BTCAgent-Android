₿ BTC Trading — Native Android App / Feasibility

Mirroring the existing BTC AI Agent web dashboard as a native Kotlin/Compose app.
TL;DR — Highly feasible. Backend already exposes a clean REST + WebSocket API behind Firebase Auth. The Android app is a thin native client over that API — no backend rewrite. Main work: rebuild the SPA's ~16 pages as Compose screens + wire Firebase Google sign-in to mint the bearer token the API already expects. Main bottlenecks: live-trading risk on mobile, background limits, and a couple of OAuth/redirect flows that don't map cleanly to native.

4 Bottlenecks / risks

  • Live-money execution on a phone — fat-finger / lost network mid-order. Mitigate: biometric gate, explicit confirm, default paper mode, server-side order idempotency.
  • FCM push not built yet — alerts go to Telegram/email today. Need device-token register endpoint + FCM send path (small backend add).
  • Android background limits — OS kills background sockets. Live WS only foreground; rely on FCM when backgrounded.
  • Pepperstone OAuth redirect — web-redirect callback needs a mobile deep-link bridge + whitelisted callback URL.
  • Masked secrets — GET settings returns ****; never display/resend masked values as real.
  • Allowed-email gate — handle access/check 403 with a clean "pending approval" screen.
  • Token expiry (~1h) — interceptor must refresh on 401 and retry.
  • Chart parity — heatmap / volume-profile / OI visuals are non-trivial natively; ship simplified first.
  • API shaped for the SPA — no mobile contract/versioning yet; consider /api/mobile/* if web & app diverge later.

5 Suggested phasing

Phase 0
Auth + allowed-email gate + live price — proves the whole pipe end-to-end.
Phase 1
Read-only: dashboard, positions, reports, scanner, briefing, regime/OI/Markov views.
Phase 2
Control: start/stop scanner, settings, manual entries (paper-first), confirm UX.
Phase 3
Live trading: execution + biometric gate + FCM alerts (needs device-token backend add).
Phase 4
Admin panel, Pepperstone OAuth, advanced charts.
